GPU Virtualization & Partitioning Tool

Overview

A streamlined PowerShell tool for creating and managing Hyper-V virtual machines with integrated GPU partitioning and driver injection. This unified solution consolidates VM creation, GPU resource allocation, and NVIDIA driver management into a single, efficient terminal interface, democratizing GPU virtualization beyond enterprise-exclusive solutions.

Core Capabilities

  • Automated VM creation with optimized Generation 2 configurations and TPM 2.0 support for Windows 11
  • Configurable GPU resource allocation (1-100% per VM) with intelligent partition value calculations
  • Direct NVIDIA driver installation into VM disk images without manual intervention
  • Driver synchronization system to keep VM drivers aligned with host GPU driver versions
  • Interactive terminal UI with color-coded logging (INFO, SUCCESS, WARN, ERROR) and real-time feedback
  • Complete VM lifecycle management from creation through configuration and maintenance
  • Smart partition detection with automatic mounting and safe cleanup procedures
  • Hardware-agnostic design supporting consumer and professional NVIDIA GPUs

Technical Architecture

Built entirely in PowerShell with direct integration to Hyper-V management APIs. Implements sophisticated GPU partition calculations using mathematical formulas to optimize VRAM, encode/decode, and compute resource allocation. The driver injection system safely mounts VM virtual hard disks, locates NVIDIA driver repositories from the host system, and copies drivers with proper file permissions.

The unified interface presents six main operations: basic VM creation, GPU partition creation, driver injection, complete GPU setup, driver updates, and safe exit. Each operation includes comprehensive error handling, user confirmation prompts for destructive actions, and detailed logging with timestamps. Auto-elevation ensures administrator privileges without manual UAC intervention.

Workflow & Best Practices

The recommended workflow starts with complete GPU setup to create a VM with immediate GPU partition configuration. After installing the operating system in the VM, users run the driver injection process to copy NVIDIA drivers from the host. The tool ensures VMs are properly shut down before partition and driver operations, preventing corruption and ensuring reliability.

Driver maintenance is streamlined through the update function, which synchronizes VM drivers with host versions after GPU driver updates. This ensures optimal performance and compatibility between host and virtualized GPU access. The tool supports both test mode with predefined values for quick deployment and custom configuration for production environments.

Impact & Applications

This tool enables GPU-accelerated virtual machines for gaming, machine learning, CAD workstations, and graphics-intensive applications without requiring expensive enterprise GPU solutions like NVIDIA GRID. It democratizes GPU virtualization technology by making it accessible on consumer hardware, opening possibilities for remote gaming rigs, multi-user systems, and development environments with GPU acceleration.

By automating complex Hyper-V and GPU partition configurations, the tool reduces setup time from hours to minutes and eliminates common configuration errors. The driver synchronization feature ensures long-term maintainability as GPU drivers are updated on the host system.

Peer-to-Peer File Sharing Platform

Overview

A decentralized web-based file sharing platform that leverages WebRTC technology to enable direct peer-to-peer transfers between browsers without uploading files to a server. This approach ensures privacy, eliminates storage costs, and enables high-speed transfers limited only by users' network connections.

Core Architecture

  • WebRTC data channels for direct browser-to-browser communication without intermediary storage
  • Flask-SocketIO signaling server for peer discovery and connection establishment
  • Room-based architecture allowing multiple users to join temporary file sharing sessions
  • STUN server integration for NAT traversal and connection establishment
  • Chunked file transfer system (16KB chunks) for handling large files efficiently
  • Real-time progress tracking with transfer speed calculations and ETA displays
  • Automatic peer connection management with reconnection handling
  • Responsive Tailwind CSS interface with drag-and-drop file selection

Technical Implementation

The backend uses Flask with Flask-SocketIO to provide a lightweight signaling server that facilitates WebRTC peer connections. When users create or join rooms, the server assigns unique room IDs and manages participant lists. The signaling server handles ICE candidate exchange and SDP offer/answer negotiation required for WebRTC connections, but never touches the actual file data.

The frontend JavaScript implementation manages RTCPeerConnection objects for each connected peer, establishing dedicated data channels for file transfers. Files are read using the FileReader API, split into 16KB chunks, and transmitted through encrypted WebRTC data channels. The receiving end reassembles chunks using ArrayBuffers and creates downloadable Blob objects. Progress tracking monitors bytes transferred and calculates real-time transfer speeds.

Security & Privacy

All file transfers occur directly between peers using WebRTC's built-in DTLS encryption, ensuring end-to-end security. The signaling server only sees connection metadata and room information—never file contents or names. Files never pass through or get stored on the server, eliminating server-side data breach risks. Room IDs are randomly generated UUID-based identifiers, providing secure session isolation between different file sharing sessions.

User Experience Features

  • Intuitive room creation and joining system with shareable room IDs
  • Real-time participant count and connection status indicators
  • Drag-and-drop file selection with multi-file support
  • Live transfer progress bars with speed and ETA calculations
  • Automatic file downloads on the receiving end using browser download APIs
  • Visual notifications for successful transfers and error conditions
  • One-click room ID copying for easy sharing with other users
  • Responsive design working seamlessly across desktop and mobile browsers

Technical Advantages

The P2P architecture eliminates server storage requirements and associated costs, making the platform highly scalable without infrastructure expenses. Direct peer connections enable transfer speeds limited only by users' upload/download bandwidth rather than server capacity. The decentralized approach ensures privacy since files never touch third-party servers, making it ideal for sensitive document sharing.

WebRTC's NAT traversal capabilities using STUN servers enable connections even when users are behind routers or firewalls. The chunked transfer system ensures reliable delivery of large files with progress tracking, while the room-based system allows for flexible multi-user collaboration scenarios.

Use Cases & Applications

This platform is ideal for quick file sharing between colleagues without email attachments or cloud storage uploads. Perfect for transferring large media files, project assets, or documents where privacy is paramount. The room-based system supports team collaboration scenarios where multiple participants need to exchange files in real-time. No account creation or file size limits make it accessible for casual users needing immediate file transfers.
